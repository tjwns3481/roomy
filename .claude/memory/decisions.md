# Architecture Decisions (ADR)

## ADR-001: Next.js Fullstack 선택
- **날짜**: 2026-01-19
- **상태**: 승인됨
- **컨텍스트**: 백엔드와 프론트엔드 분리 vs 통합
- **결정**: Next.js 14 App Router로 풀스택 구현
- **이유**:
  - 배포 단순화 (Vercel)
  - 서버 컴포넌트로 성능 최적화
  - 개인 프로젝트에 적합한 복잡도

## ADR-002: Clerk 인증 선택
- **날짜**: 2026-01-19
- **상태**: 승인됨
- **컨텍스트**: 자체 인증 vs 외부 인증 서비스
- **결정**: Clerk 사용
- **이유**:
  - 빠른 구현 (소셜 로그인 포함)
  - 보안 부담 감소
  - Next.js 통합 우수

## ADR-003: Supabase 선택
- **날짜**: 2026-01-19
- **상태**: 승인됨
- **컨텍스트**: 로컬 PostgreSQL vs 클라우드 DB
- **결정**: Supabase 사용
- **이유**:
  - 무료 티어 충분
  - Storage 기능 포함
  - 실시간 구독 가능성
