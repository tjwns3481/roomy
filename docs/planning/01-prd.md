## 소크라테스 대화에서 발견한 것들

### 진짜 욕망
- **사업으로 성장**: MRR ₩100만 이상 (6개월 후 유료 사용자 약 101명)

### 숨겨진 불확실성
- 타겟 사용자(한국 에어비앤비 호스트)를 직접 만나본 적 없음
- 유료화 전략이 불명확 (AI가 Nice-to-have인데 유료화 동기?)
- 핵심 가설들이 검증되지 않음

### 해결해야 할 문제
- 한국 호스트의 실제 페인 포인트 검증
- 유료화 포인트 재정의 필요

## 1. 문제 정의

### 1.1 현재 상황 (As-Is) **[가설 - 검증 필요]**

에어비앤비 호스트들은 마치 **매번 같은 질문에 대답하는 콜센터 직원**처럼 반복적인 게스트 문의에 시달리고 있습니다:
- "와이파이 비밀번호가 뭐예요?"
- "TV 리모컨 어떻게 써요?"
- "체크아웃 시간이 언제예요?"

종이 안내서는:
- 업데이트하려면 숙소에 직접 가야 함
- 게스트가 찾지 않거나 읽지 않음
- 비위생적이고 관리가 어려움

> **검증이 필요한 가설:**
> - 게스트의 80%는 체크인 후 1시간 내에 3가지 정보만 찾는다
> - 호스트 문의의 70%는 안내서에 이미 있는 정보다

### 1.2 목표 상황 (To-Be)

호스트가 5분 만에 디지털 안내서를 만들고, QR코드 하나로 게스트에게 공유합니다.
- 게스트: QR 스캔 → 즉시 정보 접근 → 원클릭 복사
- 호스트: 문의 감소 → 관리 시간 절약

### 1.3 왜 지금인가? (Why Now)

- 경쟁사(Hostfully, Touch Stay)가 한국어 미지원
- 국내 에어비앤비 호스트 약 4.5만명, 전체 숙박공유 호스트 15만명
- SaaS 구독 모델에 익숙해진 한국 시장

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (온보딩/로그인)
- [ ] 호스트로서, Google/이메일로 간편하게 가입할 수 있어야 한다. 왜냐하면 진입장벽을 낮춰야 하니까.
- [ ] 호스트로서, 로그인 상태가 유지되어야 한다. 왜냐하면 자주 접속하기 때문이다.

### FEAT-1: 안내서 생성 및 발행 (MVP)
- [ ] 호스트로서, 블록을 드래그 앤 드롭으로 안내서를 만들 수 있어야 한다. 왜냐하면 쉽게 만들어야 쓰니까.
- [ ] 호스트로서, 텍스트/이미지/복사 블록을 추가할 수 있어야 한다. 왜냐하면 와이파이 비번 같은 정보를 쉽게 공유해야 하니까.
- [ ] 호스트로서, 안내서를 발행하고 고유 URL을 받을 수 있어야 한다. 왜냐하면 게스트에게 공유해야 하니까.
- [ ] 호스트로서, QR코드를 다운로드할 수 있어야 한다. 왜냐하면 숙소에 부착해야 하니까.

### FEAT-2: 게스트 안내서 열람 (MVP)
- [ ] 게스트로서, 앱 설치 없이 URL/QR로 안내서를 볼 수 있어야 한다. 왜냐하면 귀찮으면 안 보니까.
- [ ] 게스트로서, 와이파이 비번 등을 원클릭 복사할 수 있어야 한다. 왜냐하면 직접 타이핑하기 싫으니까.
- [ ] 게스트로서, 모바일에서 편하게 볼 수 있어야 한다. 왜냐하면 대부분 스마트폰으로 접근하니까.

### FEAT-3: AI 챗봇 (Phase 2 - Out of scope)
- [ ] 게스트로서, 안내서에서 바로 AI에게 질문할 수 있어야 한다.
- [ ] 호스트로서, 미해결 질문 알림을 받을 수 있어야 한다.

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- FEAT-0: 기본 온보딩/로그인 (Clerk 사용)
- FEAT-1: 안내서 생성/발행 (텍스트, 이미지, 복사, 구분선 블록)
- FEAT-2: 게스트 안내서 열람 (모바일 최적화)
- QR코드 생성 및 다운로드

### 5.2 옵션 (시간 허용 시)
- 테마 커스터마이징 (색상, 폰트)
- 아코디언 블록 (FAQ용)
- 링크 블록

### 5.3 Non-goals (명시적 제외)
- **AI 챗봇**: Phase 2로 이동. Nice-to-have이며, 유료화 포인트로서의 가치 재검토 필요
- **다국어 지원**: Phase 2 이후
- **멀티 숙소 관리**: Phase 3
- **팀 협업 기능**: Phase 3
- **예약 연동 (에어비앤비 API)**: Phase 3
- **회원가입 없는 사용**: 검토했으나 데이터 관리를 위해 MVP에서는 회원가입 필수

## 7. 해결 가설과 실험

### 7.1 가설
> 만약 한국 에어비앤비 호스트에게 쉬운 디지털 안내서 도구를 제공하면,
> 그들이 월 ₩9,900을 지불하고 사용할 것이다.
> 왜냐하면 반복 문의 감소와 시간 절약의 가치가 있기 때문이다.

**검증 필요**: 이 가설은 아직 검증되지 않았습니다.

### 7.2 실험 루프

| 단계 | 내용 |
|------|------|
| 실험 1 | 한국 호스트 커뮤니티에서 사용자 인터뷰 (5-10명) |
| 관측 | 페인 포인트 확인, 지불 의향 확인 |
| 학습 | 진짜 문제인지, 얼마나 지불할 의향이 있는지 |
| 다음 | 검증되면 MVP 개발, 아니면 피봇 |

> - 진짜 유료화 동기가 무엇인지 사용자 조사에서 확인 필요
