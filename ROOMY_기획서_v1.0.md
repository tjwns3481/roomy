# Roomy 서비스 기획서 v1.0

> 에어비앤비 호스트를 위한 디지털 숙소 안내서 플랫폼

**작성일**: 2026-01-19
**버전**: 1.0
**서비스명**: Roomy (루미)

---

## Executive Summary

**한 줄 정의**: 종이 안내서를 대체하는 QR/URL 기반 모바일 웹 숙소 가이드 서비스

**핵심 가치 제안**:
- 호스트: 안내서 제작/관리 시간 80% 절감, 브랜딩 강화
- 게스트: 즉시 접근 가능한 디지털 안내서, AI 챗봇으로 24시간 응답

**타겟 시장**: 국내 에어비앤비/숙박 플랫폼 호스트 약 15만명

## 1.3 사용자 여정 맵 (As-Is vs To-Be)

### 게스트 여정: 와이파이 연결하기

| 단계 | As-Is (종이 안내서) | To-Be (Roomy) |
|------|---------------------|---------------|
| 1. 발견 | 테이블 위 안내서 찾기 (2분) | QR 스캔 or URL 클릭 (3초) |
| 2. 접근 | 여러 페이지 넘기기 (1분) | 검색 or 목차 클릭 (5초) |
| 3. 확인 | 비번 읽고 직접 입력 (30초) | 원클릭 복사 → 붙여넣기 (3초) |
| 4. 실패시 | 호스트에게 연락 (대기 ?) | AI 챗봇에 질문 (즉시) |
| **총 시간** | **3분 30초 + α** | **11초** |

### 호스트 여정: 안내서 업데이트

| 단계 | As-Is (종이 안내서) | To-Be (Roomy) |
|------|---------------------|---------------|
| 1. 수정 | Word/한글 파일 찾기 | 대시보드 접속 |
| 2. 편집 | 내용 수정 | 블록 드래그&드롭 |
| 3. 인쇄 | 프린트샵 방문 or 직접 출력 | 저장 버튼 클릭 |
| 4. 교체 | 숙소 방문해서 교체 | 자동 반영 (방문 불필요) |
| **총 시간** | **2시간~반나절** | **5분** |

| 2 | 안내서 업데이트/교체 번거로움 | ★★★★☆ | ★★★★☆ | ★★★★★ |
| 3 | 새벽/업무 중 문의 | ★★★☆☆ | ★★★★★ | ★★★★☆ |
| 4 | 멀티 숙소 관리 어려움 | ★★☆☆☆ | ★★★☆☆ | ★★★★☆ |
| 5 | 브랜딩/차별화 어려움 | ★★☆☆☆ | ★★☆☆☆ | ★★★★★ |

### 사용자 행동 가설

> **가설 1**: 게스트의 80%는 체크인 후 1시간 내에 안내서의 3가지 정보만 찾는다
> - 와이파이 비밀번호
> - TV/OTT 사용법
> - 체크아웃 시간

> **가설 2**: 호스트 문의의 70%는 안내서에 이미 있는 정보이다

> **가설 3**: QR코드 스캔율은 눈에 잘 띄는 위치 + 안내 문구가 있을 때 3배 높아진다

| 게스트 뷰 | 게스트가 보는 안내서 열람 화면 |
| 호스트 | Roomy 서비스의 주 사용자 (숙소 운영자) |
| 게스트 | 숙소 이용자 (안내서 열람자) |
